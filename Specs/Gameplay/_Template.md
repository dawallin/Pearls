# Gameplay Spec Template

## Purpose

Describe the feature, mechanic, level, or player-facing slice this spec covers.

Keep it specific. If the rule is repo-wide, it probably belongs in `Specs/Foundation/`.

## Status

Examples:

- draft
- accepted
- superseded

## Scope

State what is in scope and what is not.

## Player-Visible Behavior

Describe what the player should observe.

Focus on outcomes, not implementation.

## Rules

List the explicit mechanic or content rules.

Prefer short unambiguous statements.

## Acceptance Criteria

List the conditions that must be true for the spec to be considered implemented.

These should be testable.

## Deterministic Implications

Call out anything the core must represent explicitly, for example:

- commands
- state transitions
- event emission
- ordering rules

If there are no special implications yet, say so.

## Level Or Content Notes

If this spec depends on a test level or game level, name it here.

If not applicable, say so.

## Required Tests

List the intended proof surface:

- unit
- integration
- behavior

Be explicit about what must be covered.

## Open Questions

List unresolved product or architecture questions.

If none, say `none`.
